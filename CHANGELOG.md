# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added

- `tests/` directory with 40 unit tests covering the typed event factory,
  enums, `EventSource` parsing, `Group` / `Zone` / `Apartment` predicates,
  `CombinedCondition` factories, and frozen-dataclass immutability.
- GitHub Actions CI pipeline (`ruff`, `mypy --strict`, `pytest` on
  Python 3.10/3.11/3.12).
- Dependabot configuration (weekly pip, monthly github-actions).
- Issue templates (bug report, feature request) and PR template.
- `CONTRIBUTING.md` and `SECURITY.md`.
- `pydigitalstrom/py.typed` PEP 561 marker.

### Changed

- Typed `AlarmEngine.events_since` as a generic overload so call sites
  filtering by an `event_type` get the narrowed return type without
  `cast`.

## [0.1.0] - 2026-05-30

### Added

- Initial release of the `pydigitalstrom` async Python library for the
  digitalSTROM dSS REST API.
- **Authentication**: `AppToken` + `SessionManager` with automatic
  re-login on session-token expiry.
- **HTTP client**: `DssClient` with auto-retry on 401 and the long-poll
  helper `event_long_poll`.
- **Apartment inventory**: `fetch_apartment` returning fully populated
  `Apartment` / `Zone` / `Cluster` / `Group` / `Device` / `Sensor` /
  `BinaryInput` / `Circuit` frozen dataclasses.
- **Event stream**: `EventStream` async-iterator with 13 typed event
  classes (`CallSceneEvent`, `UndoSceneEvent`, `ButtonClickEvent`,
  `StateChangeEvent`, `DeviceBinaryInputEvent`, `DeviceSensorEvent`,
  `ZoneSensorValueEvent`, `ZoneSensorErrorEvent`, `HighLevelEvent`,
  `DeviceEvent`, `ExecutionDeniedEvent`, `RunningEvent`,
  `ModelReadyEvent`). Auto-reconnect with exponential backoff,
  auto re-subscribe on `RunningEvent`.
- **Event dispatcher**: `EventDispatcher` for handler-registration-style
  consumption.
- **Circuit power monitoring**: `CircuitMonitor` with per-dSM watts +
  energy readings and `CircuitState` classification (`NORMAL`,
  `POWERING_UP`, `OFFLINE`, `UNREACHABLE`, `VIRTUAL`). Detection rules
  empirically verified against a real box.
- **Alarm framework**: `AlarmEngine` orchestrating circuit polling,
  event stream, and state polling. Built-in rules: `CircuitOutageRule`,
  `MotionWhileAbsentRule`, `ZoneTemperatureLowRule`, `SensorOutageRule`,
  `UserStateAlarmRule`. Bring your own `AlarmRule` for anything else.
- **Property-tree walker**: `PropertyTreeWalker` with type-aware
  reads/writes.
- **System-addon CRUD** for all four user-facing addons with verified
  save protocols (reverse-engineered via DevTools):
  - `TimedEventsAddon` (Zeitschaltuhr) — list / get / save / delete
  - `UserActionsAddon` — list / get / find-by-name / trigger-by-name /
    trigger-by-id / save / delete
  - `SceneResponderAddon` — list (with `include_system` filter) / get /
    save / delete / set_enabled
  - `UserStatesAddon` — full CRUD across all four user-creatable
    sub-categories: `custom-states`, `combined-states`,
    `triggered-states`, `zone-sensor-states`. Read-only support for
    `device-sensor-states` (those are auto-generated by other addons).
- **CLI tool**: `pydigitalstrom-cli` with `health-check`, `apartment`,
  `list-zones`, `list-zones-all`, `list-devices`, `list-circuits`,
  `call-scene`, `undo-scene`, `watch-events`, `raw-get`, `property-tree`.
- **Type hints**: PEP 561 `py.typed` marker. The full public API
  passes `mypy --strict`.
- **Documentation**:
  - `docs/LIBRARY_DESIGN.md` — full public-API reference.
  - `docs/ARCHITECTURE.md` — layers, event flow, reconnect strategy.
  - `docs/DSS_API_NOTES.md` — reverse-engineered dSS REST API notes
    including all the protocol quirks we found (`addonid` vs
    `addon-id`, `.config` vs `-config` event naming, action-placeholder
    pattern, sequential vs timestamp IDs, etc.).
